## 1. What the app does

CHASERMap provides a public-facing, searchable view of Colorado campaign finance data sourced from [TRACER](https://tracer.sos.colorado.gov), the Colorado Secretary of State's campaign finance reporting system.

**Features:**
- **Unified search** — find any race, candidate, or city instantly from the homepage
- **Interactive district map** (`/map/`) — choropleth map colored by fundraising margin, cash on hand, competitiveness, burn rate, or loan reliance
- **Race pages** (`/races/<slug>/`) — all candidates in a district with fundraising stats and a list of cities in the district
- **Candidate pages** (`/candidates/<slug>/`) — individual candidate financials, committee info, and incumbent status
- **Finance Explorer** (`/map/finance.html`) — donor lookup, quarterly fundraising timelines (Chart.js), and itemized expenditure lists

## 3. Architecture overview

```
┌──────────────────────────────────────────────────────────────────┐
│  Data Sources (external)                                         │
│  tracer.sos.colorado.gov    US Census Bureau    CO Legislature   │
└─────────────┬───────────────────────┬──────────────────┬────────┘
              │ Playwright scrape      │ HTTP download    │ Excel
              ▼                        ▼                  ▼
┌─────────────────┐      ┌────────────────────┐   ┌───────────────┐
│   scraper.py    │      │     build.py        │   │ co_legislators│
│                 │      │  (library funcs)    │   │    .xlsx      │
│  → tracer_*.csv │      │  shapefile_to_geojson│  └───────┬───────┘
└────────┬────────┘      │  load_places        │          │
         │               │  build_city_map     │◄─────────┘
         │               │  load_races         │◄── tracer_*.csv
         │               └──────────┬──────────┘
         │                          │
         ▼                          ▼
┌────────────────────────────────────────────┐
│             ingest.py                       │
│  Reads build.py functions, writes SQLite    │
│  → data/chaser.db                           │
└───────────────────────┬────────────────────┘
                        │
                        ▼
              ┌──────────────────┐
              │     app.py       │
              │  Flask web app   │◄── Jinja2 templates/
              │  queries chaser.db│   static/
              └──────────────────┘

Separate static generation:
  build.py main()          → map/index.html  (Leaflet map)
  finance_builder.py main()→ map/finance.html (Finance Explorer)
```

**Key design decision:** `app.py` does not import `ingest.py` and does not run the data pipeline at startup.  Data must be pre-loaded into `chaser.db` by running `ingest.py` separately.  This keeps the web server fast and stateless.

## 4. Data pipeline

Run these steps in order whenever you want to refresh the data:

### Step 1 — Download candidate listings from CO SOS

Go to [tracer.sos.colorado.gov](https://tracer.sos.colorado.gov) → Candidate Search.
Filter by 2026, download the CSV for:
- Colorado Senate → save as `data/rpt_CF_CAND_001.csv`
- Colorado House → save as `data/rpt_CF_CAND_001-2.csv`
- Statewide offices → save as `data/rpt_CF_CAND_001-statewide.csv`

These files provide **party affiliation**, **committee name**, and **listing status** — not available in the race search export.

### Step 2 — Scrape TRACER financial data

```bash
# Install Playwright browser (first time only)
playwright install chromium

# Scrape all Senate + House districts
python3 scraper.py

# Scrape statewide executive offices (Governor, AG, etc.)
python3 scraper.py --statewide
```

Outputs:
- `data/tracer_2026_all_districts.csv`
- `data/tracer_2026_statewide.csv`

If the candidate listing files change but you don't need new TRACER financial data, use:
```bash
python3 scraper.py --reprocess
```

### Step 3 — Load into SQLite

```bash
python3 ingest.py
```

This creates (or updates) `data/chaser.db`.  Re-running is safe — existing rows for the target year are deleted and re-inserted.

### Step 4 — Regenerate static map HTML (optional)

```bash
python3 build.py
```

Outputs `map/index.html`.  The GeoJSON shapefiles and place centroids are downloaded from the US Census on the first run and cached locally — subsequent runs are fast.

### Step 5 — Regenerate Finance Explorer (optional)

```bash
python3 finance_builder.py
```

Reads `data/finances/2026_*.csv` and outputs `map/finance.html`.

## 5. Database schema

The Flask app reads from `data/chaser.db` (SQLite).

### `candidates` table

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER PK | Auto-increment |
| `year` | INTEGER | Election year (e.g. 2026) |
| `chamber` | TEXT | "Senate", "House", or office name for statewide |
| `district` | TEXT | District number ("3") or race slug for statewide |
| `race_slug` | TEXT | URL slug, e.g. "senate-3" or "governor" |
| `label` | TEXT | Human-readable label, e.g. "Senate District 3" |
| `name` | TEXT | TRACER format: "LAST, FIRST MIDDLE" |
| `party` | TEXT | "Democratic", "Republican", "Unaffiliated", etc. |
| `committee` | TEXT | Campaign committee name |
| `status` | TEXT | "Active" or "Terminated" |
| `raised` | REAL | Total monetary contributions received |
| `spent` | REAL | Total monetary expenditures |
| `coh` | REAL | End-of-period cash on hand |
| `beg` | REAL | Beginning-of-period cash on hand |
| `loans` | REAL | Total self-loans received |
| `vsl` | TEXT | "Y"/"N" — accepted voluntary spending limit |
| `incumbent` | INTEGER | 1 if current seat-holder, 0 otherwise |
| `slug` | TEXT | URL slug for candidate detail page |

### `city_map` table

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER PK | Auto-increment |
| `year` | INTEGER | Election year |
| `chamber` | TEXT | "Senate" or "House" |
| `district` | TEXT | District number string |
| `city` | TEXT | City/place name (from Census TIGER data) |

### Indexes

- `idx_cand_race` — on `(year, race_slug)` — used by race detail page
- `idx_cand_slug` — on `slug` — used by candidate detail page
- `idx_city_lookup` — on `(year, chamber, district)` — used by race page city list

## 6. Key data sources

| Source | What it provides | How accessed |
|--------|-----------------|--------------|
| TRACER race search | Financial figures: raised, spent, CoH, loans, beginning balance, VSL | Scraped by `scraper.py` via Playwright |
| TRACER candidate search | Party, committee name, listing status | Manual CSV download → `data/rpt_CF_CAND_001*.csv` |
| TRACER finance exports | Individual contributions, expenditures, loans | Manual CSV download → `data/finances/` |
| US Census GENZ2022 | Colorado district boundary shapefiles | Downloaded by `build.py` via `urllib` |
| US Census TIGER2022 | Colorado incorporated places (for city mapping) | Downloaded by `build.py` via `urllib` |
| CO Legislature Excel | Current legislators for incumbent detection | Manual download → `data/co_legislators.xlsx` |

## 7. Running locally

### Prerequisites

- Python 3.11+
- pip

### Setup

```bash
# 1. Clone the repo
git clone <repo-url>
cd CHASERMap

# 2. Install Python dependencies
pip install -r requirements.txt

# 3. Install Playwright's Chromium browser (needed for scraper only)
playwright install chromium

# 4. Ensure data/chaser.db exists (skip if you already have it)
python3 ingest.py

# 5. Start the dev server
python3 app.py
```

Open [http://localhost:5001](http://localhost:5001) in your browser.

### Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | `5001` | Port for the Flask dev server |

## 8. Updating data for a new scrape cycle

Run these in order:

```bash
# Re-scrape TRACER (takes ~10-20 min for all districts)
python3 scraper.py
python3 scraper.py --statewide

# Reload into SQLite
python3 ingest.py

# Rebuild static maps (optional)
python3 build.py
python3 finance_builder.py
```

If you only updated the candidate listing CSVs (not the financial data):
```bash
python3 scraper.py --reprocess
python3 ingest.py
```

## 9. Adapting for a new election year

When starting a new election cycle (e.g. 2028), update these locations:

| File | Location | Change |
|------|----------|--------|
| `app.py` | `CURRENT_YEAR = 2026` | Update to new year |
| `ingest.py` | `DEFAULT_YEAR = 2026` | Update to new year |
| `scraper.py` | `OUTPUT_FILE`, `STATEWIDE_OUTPUT_FILE`, `ELECTION_VALUE` | Update filenames and discover new election value |
| `build.py` | `CSV_FILE`, `STATEWIDE_CSV`, HTML title strings | Update filenames |
| `finance_builder.py` | `TRACER_LEGISLATIVE`, `TRACER_STATEWIDE`, `CONTRIBUTIONS_CSV`, etc. | Update filenames |
| `data/` | Replace `*.csv` files with new cycle data | New data |
| `data/co_legislators.xlsx` | Download new roster from legislature website | New roster |

Also run `python3 scraper.py --discover` to check whether TRACER's election dropdown value has changed.

## 10. Deploying to Render

[Render](https://render.com) can host the Flask app on its free tier.  A `render.yaml` file is included in this repo for one-click configuration.

### What `render.yaml` configures

- **Build command:** installs Python dependencies
- **Start command:** runs Gunicorn with the Flask app
- **Environment variables:** sets `PYTHON_VERSION`
- **Health check:** pings `/` to verify the app started successfully

### Deploy steps

1. **Push this repo to GitHub** (if not already there)

2. **Create a Render account** at [render.com](https://render.com) (free)

3. **Create a new Web Service:**
   - Click **New → Web Service**
   - Connect your GitHub account and select this repository
   - Render will detect `render.yaml` automatically and pre-fill all settings

4. **Review settings:**
   - **Name:** choose a service name (e.g. `chaser-map`)
   - **Region:** US East or US West (free tier)
   - **Branch:** `main` (or whichever branch you want to deploy)
   - All other settings come from `render.yaml`

5. **Click Deploy** — the first build takes ~2-3 minutes

6. **Verify:** visit your Render URL (e.g. `https://chaser-map.onrender.com`)

### Important: the SQLite database

The SQLite file `data/chaser.db` **must be committed to the repository** for the app to work on Render's free tier.  Render's free tier does not provide persistent disk storage, so any files written at runtime are lost on redeploy.

Before deploying, run locally:
```bash
python3 ingest.py
git add data/chaser.db
git commit -m "Update chaser.db with latest data"
git push
```

Similarly, commit `map/index.html` and `map/finance.html` if you've regenerated them:
```bash
python3 build.py
python3 finance_builder.py
git add map/index.html map/finance.html
git commit -m "Regenerate map and finance HTML"
git push
```

### Custom domain

To use a custom domain (e.g. `chaser.chasecrom.com`):
1. In the Render dashboard, go to your service → **Settings → Custom Domains**
2. Add your domain and follow the DNS instructions
3. Update `CNAME` in the repo root to match your domain (legacy reference — Render does not use this file)

### Free tier limitations

| Limitation | Details |
|------------|---------|
| Sleep after inactivity | Free web services spin down after ~15 min of no traffic; first request after sleep takes ~30 seconds |
| No persistent disk | All runtime file writes are ephemeral — commit your data files to git |
| 512 MB RAM | Sufficient for this app; `build.py` and `scraper.py` should be run locally |
| Bandwidth | 100 GB/month outbound (generous for this use case) |

> **Tip:** For a production deployment that stays awake, upgrade to Render's Starter tier ($7/month) or add a cron job / uptime monitor to ping the app every 10 minutes.

## 11. Troubleshooting

### `ModuleNotFoundError: No module named 'shapefile'`
```bash
pip install pyshp
```
The package name is `pyshp` but it imports as `shapefile`.

### `FileNotFoundError: data/chaser.db`
The database hasn't been built yet.  Run:
```bash
python3 ingest.py
```

### `FileNotFoundError: data/tracer_2026_all_districts.csv`
The TRACER scrape hasn't been run.  Follow [Step 2 in the data pipeline](#step-2--scrape-tracer-financial-data).

### Scraper fails with "District validation failed — filter state lost"
The ASP.NET session state was lost between requests.  The scraper automatically attempts to recover by re-navigating to TRACER and re-applying the filters.  If recovery also fails, re-run the scraper — it will skip districts it already collected.

### Incumbent detection is wrong for a candidate
Check `build.py`:
- `_NICKNAME_FIRST_NAMES` — add an entry if the candidate files under a different first name
- `_EXTRA_INCUMBENTS` — add an entry if the candidate is missing from `co_legislators.xlsx`
- Refresh `co_legislators.xlsx` from the [CO legislature website](https://leg.colorado.gov/legislators)

### The map (`/map/`) shows no data
`map/index.html` is a static file generated by `build.py`.  Re-run:
```bash
python3 build.py
```

### Finance Explorer shows no candidates
`map/finance.html` is generated by `finance_builder.py`.  Ensure `data/finances/*.csv` files exist, then re-run:
```bash
python3 finance_builder.py
```
